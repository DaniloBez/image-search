# AI Image Search Engine

**AI Image Search Engine** — це інтелектуальна система пошуку зображень, яка використовує локальні нейромережі (Vision LLM) для автоматичного розпізнавання контенту на фото та класичний алгоритм TF-IDF для швидкого та релевантного текстового пошуку.

---

## Головні фічі (Features)

* **Машинний зір (Computer Vision):** Автоматична генерація детальних описів для кожного завантаженого зображення за допомогою локальної моделі ШІ (Ollama / LLaVA).
* **Smart Search (TF-IDF):** Швидкий лексичний пошук на основі інвертованого індексу та алгоритму TF-IDF з ранжуванням результатів за релевантністю.

---

## Технологічний стек

**Backend:**
* Java 25
* Spring Boot 3 (Web, Data JPA)
* PostgreSQL 
* Swagger / OpenAPI 3 (для документації API)

**AI & Search:**
* [Ollama](https://ollama.com/) (для локального запуску LLM)
* Модель: `llava`
* Алгоритм: Власна реалізація TF-IDF (Term Frequency - Inverse Document Frequency)

**Frontend:**
* HTML5, CSS3 
* Vanilla JavaScript 
* Thymeleaf

## Системні вимоги (System Requirements)

Оскільки проєкт розгортає локальну нейромережу (модель LLaVA 7B) у Docker-контейнері, переконайтеся, що ваш пристрій відповідає таким вимогам:

* **Мінімальні (лише CPU):** 8 GB RAM (обробка та опис одного зображення може займати від 1 до 2 хвилин).
* **Рекомендовані (GPU / Apple Silicon):** 16 GB RAM або більше. Наявність відеокарти (Nvidia/AMD) або чипа Apple M-серії (M1/M2/M3) значно прискорить роботу.
* **Місце на диску:** ~16 GB вільного простору (Docker-образи Postgres, Java, Ollama + сама модель LLaVA важить близько 15 GB).

## Як запустити (Getting Started with Docker)

### 1. Передумови (Prerequisites)
Переконайтеся, що на вашому комп'ютері встановлено:
* [Docker Desktop](https://www.docker.com/products/docker-desktop/) (або Docker Engine)

### 2. Запуск інфраструктури
Склонуйте репозиторій та перейдіть у папку проєкту:
```bash
git clone https://github.com/DaniloBez/image-search
cd image-search
```

Запустіть усі контейнери (База даних, Backend-сервер, Ollama) у фоновому режимі однією командою:
```bash 
docker-compose up -d
```

### Завантаження Vision-моделі (LLaVA)
Оскільки контейнер Ollama запускається "чистим", при першому запуску необхідно завантажити ШІ-модель всередину контейнера.
Виконайте цю команду в терміналі (завантаження може зайняти кілька хвилин залежно від швидкості інтернету):

```bash
docker exec -it image-search-ai ollama run llava
```

### 4. Використання додатку
Після успішного запуску всіх сервісів та завантаження моделі, додаток буде доступний за адресою:
 * http://localhost:8080

Документація API (Swagger):
 * http://localhost:8080/swagger-ui.html

### Зупинка проєкту
Щоб зупинити всі контейнери без втрати даних (зображення та індекс збережуться на Docker-вольюмах), виконайте:

```bash
docker-compose down
```
